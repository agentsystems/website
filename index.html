Why AgentSystems

Manage Your AI Infrastructure

While cloud AI services lock you into their infrastructure, AgentSystems supports self-hosted deployments. Process data locally, switch providers through configuration, and operate your own AI infrastructure.

Data Location Management

Process data within your own environment. Deploy on-premises, in your VPC, or in air-gapped networks with local model support. Intended for scenarios where data location matters.

Multiple Provider Support

Compatible with OpenAI, Anthropic, AWS Bedrock, or local models. Configuration at the platform level for provider switching. Direct API key usage supported.

Security Features

Agents run in Docker containers. Outbound requests can be routed through an HTTP CONNECT proxy with URL allowlisting. Audit trail functionality uses SHA-256 hash chaining.

How It Works

Gateway-Based Agent Orchestration

AgentSystems uses a gateway architecture. The gateway routes requests to containerized agents, logs activity to an audit trail, and manages container lifecycle.

Request arrives at gateway

HTTP request hits the control plane on port 18080, generates a unique thread ID for tracking

Gateway discovers agent

Attempts to find target agent via Docker labels, starts container if needed, checks health status

Agent processes request

Containerized environment handles the request, saves artifacts to thread-specific directory

Audit trail logging

Request, response, and metadata logging available with cryptographic hash chaining

Thread Separation

Requests receive unique thread IDs with separate artifact storage

Auto Lifecycle

Containers can start on demand and stop when idle

Progress Tracking

Progress updates available for long-running operations

Multi-Registry

Support for pulling agents from Docker registries with authentication

Quick Start

Deploy in 5 Minutes

Get AgentSystems running on your machine.

Requirements

  • Python 3.11+
  • Docker
  • macOS or Linux

What Starts

  • Control Plane
  • Web UI (port 3001)
  • PostgreSQL
# Install
curl -fsSL https://github.com/agentsystems/agentsystems/releases/latest/download/install.sh | sh

# Initialize
agentsystems init my-deployment
cd my-deployment

# Start
agentsystems up

# Pull local model
docker exec agentsystems-ollama-1 ollama pull gemma3:1b

# Open UI
open http://localhost:3001
