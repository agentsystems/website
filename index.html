Why AgentSystems

The AI Agent Dilemma: Send your data to third-party AI companies, or build everything yourself

AgentSystems offers another way: Run third-party agents on your infrastructure. Access an emerging ecosystem of community and commercial agents, processing where you control, not building from scratch.

Your Data, Your Environment

Deploy on-premises, in your VPC, or air-gapped networks. Process sensitive data where it lives.

Multiple Model Providers

OpenAI, Anthropic, AWS Bedrock, or local models via Ollama. Switch providers with configuration changes.

Container Isolation

Each agent runs in Docker with resource boundaries. HTTP proxy support with URL allowlisting for egress control.

Flexible Deployment

Laptop, server, Kubernetes - deploy anywhere Docker runs.

Observable

Langfuse integration for tracing LLM calls and agent execution.

Audit Trail

SHA-256 hash-chained logs that track operations.

Thread Isolation

Each request gets a unique thread ID with separate artifacts.

Quick Start

Deploy in 5 Minutes

Get AgentSystems running on your machine.

Requirements

  • Python 3.11+
  • Docker
  • macOS or Linux

What Starts

  • Control Plane
  • Web UI (port 3001)
  • PostgreSQL
# Install on macOS or Linux
curl -fsSL https://github.com/agentsystems/agentsystems/releases/latest/download/install.sh | sh

# Initialize
agentsystems init my-deployment
cd my-deployment

# Start
agentsystems up

# Pull local model
docker exec agentsystems-ollama-1 ollama pull gemma3:1b

# Open UI
open http://localhost:3001
