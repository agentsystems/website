Platform Architecture

Why AgentSystems?

Many AI platforms operate as cloud services. AgentSystems is a self-hosted control plane designed to run on your infrastructure. Agents run in containers, API calls can be logged, and you maintain control over the deployment.

🔒

Data Location Control

Process data within your own environment. Deploy on-premises, in your VPC, or in air-gapped networks with local model support. Designed for scenarios where data location matters.

🔄

Multiple Provider Support

Compatible with OpenAI, Anthropic, AWS Bedrock, or local models. Configuration at the platform level allows provider switching. Direct API key usage supported.

🛡️

Security Features

Agents run in Docker containers. Outbound requests can be routed through an HTTP CONNECT proxy with URL allowlisting. Audit trail functionality uses SHA-256 hash chaining.

How It Works

Platform Architecture

AgentSystems uses a gateway architecture. The gateway routes requests to containerized agents, logs activity to an audit trail, and manages container lifecycle.

1

Request arrives at gateway

HTTP request hits the control plane on port 18080, generates a unique thread ID for tracking

2

Gateway discovers agent

Attempts to find target agent via Docker labels, starts container if needed, checks health status

3

Agent processes request

Isolated container handles the request, saves artifacts to thread-specific directory

4

Audit trail logging

Request, response, and metadata logging available with cryptographic hash chaining

Thread Isolation

Requests receive unique thread IDs with separate artifact storage

Auto Lifecycle

Containers can start on demand and stop when idle

Progress Tracking

Progress updates available for long-running operations

Multi-Registry

Support for pulling agents from Docker registries with authentication

Getting Started

Getting Started

Deploy the platform with a few commands. Run your first agent shortly after.

📋 Prerequisites

  • Python 3.11+
  • Docker Desktop (Mac) or Docker Engine (Linux)
  • 4GB RAM available
  • 10GB disk space

🖥️ What You Get

  • Control plane gateway on port 18080
  • Web UI on port 3001
  • PostgreSQL for audit trails
  • HTTP proxy for egress control
# 1. Install the AgentSystems CLI
curl -fsSL https://github.com/agentsystems/agentsystems/releases/latest/download/install.sh | sh

# 2. Initialize your deployment
agentsystems init my-deployment
cd my-deployment

# 3. Start the platform
agentsystems up

# 4. Open the UI
open http://localhost:3001

# 5. (Optional) Download a local model for testing
docker exec ollama ollama pull llama3.2

Deploy Your First Agent

# Use the agent template
git clone https://github.com/agentsystems/agent-template my-agent
cd my-agent

# Build your agent container
docker build -t my-agent:latest .

# Add to your deployment
cd ../my-deployment
# Edit docker-compose.yml to include your agent

# Deploy and run
agentsystems up
agentsystems run my-agent '{"prompt": "Hello, AgentSystems!"}'
