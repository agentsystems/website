Why AgentSystems

Manage Your AI Infrastructure

Process data locally, switch providers through configuration, and maintain control over your AI operations.

Your Data, Your Environment

Deploy on-premises, in your VPC, or air-gapped networks. Process sensitive data where it lives without external dependencies.

Any Model Provider

OpenAI, Anthropic, AWS Bedrock, or local models like Ollama. Switch providers with configuration changes.

Container Isolation

Each agent runs in Docker with resource boundaries. HTTP proxy support with URL allowlisting for egress control.

Flexible Deployment

Laptop, server, Kubernetes - deploy anywhere Docker runs.

Observable

Langfuse integration for tracing LLM calls and agent execution.

Audit Trail

SHA-256 hash-chained logs that track operations.

Thread Isolation

Each request gets a unique thread ID with separate artifacts.

Quick Start

Deploy in 5 Minutes

Get AgentSystems running on your machine.

Requirements

  • Python 3.11+
  • Docker
  • macOS or Linux

What Starts

  • Control Plane
  • Web UI (port 3001)
  • PostgreSQL
# Install
curl -fsSL https://github.com/agentsystems/agentsystems/releases/latest/download/install.sh | sh

# Initialize
agentsystems init my-deployment
cd my-deployment

# Start
agentsystems up

# Pull local model
docker exec agentsystems-ollama-1 ollama pull gemma3:1b

# Open UI
open http://localhost:3001
